# Hoja de Ruta

Esta hoja de ruta describe nuestras características e mejoras planificadas para QZPay. Las prioridades pueden cambiar según los comentarios de la comunidad y los patrones de uso del mundo real.

<Aside type="note">
Esta hoja de ruta es indicativa, no un compromiso. Las características pueden agregarse, modificarse o posponerse según las necesidades de la comunidad y consideraciones técnicas.
</Aside>

## Solicitar una Característica

¿Tienes una idea para QZPay? ¡Nos encantaría escucharla!

1. **Revisar solicitudes existentes** - Buscar en [GitHub Issues](https://github.com/qazuor/qzpay/issues) solicitudes similares
2. **Abrir una discusión** - Iniciar una [GitHub Discussion](https://github.com/qazuor/qzpay/discussions) para recopilar comentarios
3. **Enviar un issue** - Crear un issue de solicitud de característica con detalles

Al solicitar una característica, incluye:
- **Caso de uso** - ¿Qué problema resuelve esto?
- **Solución propuesta** - ¿Cómo debería funcionar?
- **Alternativas consideradas** - ¿Qué otros enfoques consideraste?
- **Prioridad** - ¿Qué tan importante es esto para tu proyecto?

## Calendario de Lanzamientos

Nuestro objetivo es lanzar actualizaciones regularmente:

- **Lanzamientos de parche** (1.0.x) - Según sea necesario para corrección de errores
- **Lanzamientos menores** (1.x.0) - Cada 4-8 semanas con nuevas características
- **Lanzamientos mayores** (x.0.0) - Según sea necesario para cambios incompatibles (intentamos minimizar estos)
