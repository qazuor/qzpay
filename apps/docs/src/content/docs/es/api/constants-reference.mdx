## Estado de Suscripcion

Valores de estado para el ciclo de vida de suscripciones.

```typescript
const QZPAY_SUBSCRIPTION_STATUS = {
  ACTIVE: 'active',
  TRIALING: 'trialing',
  PAST_DUE: 'past_due',
  PAUSED: 'paused',
  CANCELED: 'canceled',
  UNPAID: 'unpaid',
  INCOMPLETE: 'incomplete',
  INCOMPLETE_EXPIRED: 'incomplete_expired'
} as const;
```

| Constante | Valor | Descripcion |
|----------|-------|-------------|
| `ACTIVE` | `'active'` | Suscripcion activa y facturando normalmente |
| `TRIALING` | `'trialing'` | En periodo de prueba gratuito |
| `PAST_DUE` | `'past_due'` | Pago fallido, en periodo de reintento |
| `PAUSED` | `'paused'` | Pausada temporalmente por cliente o administrador |
| `CANCELED` | `'canceled'` | Suscripcion ha sido cancelada |
| `UNPAID` | `'unpaid'` | Todos los reintentos de pago agotados |
| `INCOMPLETE` | `'incomplete'` | Pago inicial pendiente (3DS, etc.) |
| `INCOMPLETE_EXPIRED` | `'incomplete_expired'` | Pago inicial expirado |

### Uso

```typescript
import { QZPAY_SUBSCRIPTION_STATUS, type QZPaySubscriptionStatus } from '@qazuor/qzpay-core';

function isSubscriptionActive(status: QZPaySubscriptionStatus): boolean {
  return status === QZPAY_SUBSCRIPTION_STATUS.ACTIVE ||
         status === QZPAY_SUBSCRIPTION_STATUS.TRIALING;
}

// Check if subscription needs attention
if (subscription.status === QZPAY_SUBSCRIPTION_STATUS.PAST_DUE) {
  notifyCustomer('Payment failed, please update your payment method');
}
```

## Estado de Pago

Valores de estado para transacciones de pago.

```typescript
const QZPAY_PAYMENT_STATUS = {
  PENDING: 'pending',

  REFUNDED: 'refunded',
```

| Constante | Valor | Descripcion |
|----------|-------|-------------|
| `PENDING` | `'pending'` | Pago iniciado, esperando procesamiento |

| `SUCCEEDED` | `'succeeded'` | Pago completado exitosamente |

| `CANCELED` | `'canceled'` | Pago fue cancelado |
| `REFUNDED` | `'refunded'` | Reembolso completo emitido |

### Uso

```typescript
import { QZPAY_PAYMENT_STATUS, type QZPayPaymentStatus } from '@qazuor/qzpay-core';

function canRefund(status: QZPayPaymentStatus): boolean {
  return status === QZPAY_PAYMENT_STATUS.SUCCEEDED;
}

if (payment.status === QZPAY_PAYMENT_STATUS.DISPUTED) {
  await handleDispute(payment);
}
```

```typescript
const QZPAY_INVOICE_STATUS = {
  DRAFT: 'draft',
```

| Constante | Valor | Descripcion |

### Uso

```typescript
import { QZPAY_INVOICE_STATUS, type QZPayInvoiceStatus } from '@qazuor/qzpay-core';

// Only finalized invoices can be paid
if (invoice.status === QZPAY_INVOICE_STATUS.DRAFT) {
  await billing.invoices.finalize(invoice.id);
}

// Void unpaid invoices
if (invoice.status === QZPAY_INVOICE_STATUS.OPEN) {
  await billing.invoices.void(invoice.id);
}
```

## Modo de Apilamiento de Descuentos

Como se combinan multiples descuentos.

```typescript
const QZPAY_DISCOUNT_STACKING_MODE = {
  NONE: 'none',
  ADDITIVE: 'additive',
  MULTIPLICATIVE: 'multiplicative',
  BEST: 'best'
} as const;
```

| Constante | Valor | Descripcion |
|----------|-------|-------------|
| `NONE` | `'none'` | Solo aplica el primer descuento valido |
| `ADDITIVE` | `'additive'` | Los descuentos se suman juntos |
| `MULTIPLICATIVE` | `'multiplicative'` | Los descuentos se aplican secuencialmente |
| `BEST` | `'best'` | Aplica el mejor descuento para el cliente |

### Ejemplo

```typescript
// Original: $100
// Code 1: 20% off
// Code 2: $10 off

// ADDITIVE: $100 - $20 - $10 = $70
// MULTIPLICATIVE: $100 - $20 = $80, then $80 - $10 = $70
// With two 20% codes on $100:
// ADDITIVE: $100 - $20 - $20 = $60
// MULTIPLICATIVE: $100 - $20 = $80, then $80 - $16 = $64
```
